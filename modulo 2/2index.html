Working with fonts

Font-size

CSS offers a number of different units for expressing length. There are absolute measurements, such as cm, mm, px, in (inches), pt (point), pc (pica) and relative measurements, like em (relative to capital M width, standart body-size, 16px), ex (relative to x-height), rem (em of the root font), vw (viewport width). The most recommended units are em and px, since the others fluctuate their size much more on different screens and browsers.

Font weight, style and text decoration properties *

  • Font-weight: Dictates the thickness of the letters. The possible values are "lighter", "normal", "bold" and "bolder" or it can be indicated with hundreds between 100 and 900.
  • Font-style: Sets the font to italic style.
  • Text-decoration: Sets a styled underline, it can be solid, dotted, double, dashed, etc.

    • * You can combine all these properties in one shorthand, the "font" property. The typing order is font-style -> font-weight -> font-size -> font family.

Exporting fonts

Example of external font, Coffee Cake!

When choosing an external font from websites like DaFont and MyFonts, you need to keep an eye for font formats. They can be .otf (opentype) .ttf (truetype) .eot (embedded-opentype) .aat (truetype apple advanced typography) or .svg (scalable vetor graphics). The usability of these formats depends on browser compatibility.

To load the chosen font on your website, you need to include the font file on your web server and use the @font-face rule on CSS, which specifies the font and saves it inside the code.
